# MVP1-005 — UI

## Goal

Собрать пользовательский интерфейс MVP 1: тренер и спортсмен работают через Reps без мессенджеров, таблиц и голосовых — все сценарии доступны в личном кабинете.

## Requirements

### Кабинет тренера

- Форма отправки приглашения по email
- Список спортсменов со статусами приглашений
- Карточка спортсмена: профиль, рекорды, история feedback
- Просмотр ссылок на Garmin Connect из feedback

### Кабинет спортсмена

- Принятие / отклонение входящего приглашения
- Форма профиля: рост, вес, возраст, пол
- Управление личными рекордами (добавить / редактировать / удалить)
- Форма обратной связи по тренировке с полем ссылки Garmin
- История своих feedback

### Общее

- UI соответствует ролям из MVP 0; недоступные разделы скрыты
- Ошибки валидации отображаются рядом с полями
- Состояния загрузки и пустые списки обработаны
- Адаптивная вёрстка для desktop и mobile
- ESLint и Prettier проходят без ошибок

## Acceptance Criteria

- [x] Тренер отправляет приглашение через UI
- [x] Спортсмен принимает приглашение через UI
- [x] Спортсмен заполняет профиль и управляет рекордами через UI
- [x] Спортсмен оставляет feedback с ссылкой Garmin через UI
- [x] Тренер просматривает спортсменов и их данные через UI
- [x] Ошибки API отображаются пользователю понятно
- [x] `npm run lint` и `npm run format:check` проходят
- [x] Основные пользовательские сценарии покрыты e2e или integration UI-тестами (если инфраструктура готова)

- Backend API MVP 1 полностью работает
- Личные кабинеты по ролям из MVP 0 существуют
