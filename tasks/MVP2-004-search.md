# MVP2-004 — Search

## Goal

Дать тренеру и спортсмену чтение планов и итогов тренировок за календарные периоды: неделя и месяц вперёд (план), неделя и месяц назад/включая текущий (итоги).

> В контексте Reps «поиск» — запросы и фильтрация по датам, не полнотекстовый поиск.

## Requirements

### План тренировок

| Кто       | Endpoint (концепт)         | Период                         |
| --------- | -------------------------- | ------------------------------ |
| Тренер    | план выбранного спортсмена | неделя, месяц от `anchor_date` |
| Спортсмен | свой план                  | неделя, месяц от `anchor_date` |

- Ответ: список `PlannedWorkout` с типом, циклами и упражнениями
- Пустые дни в периоде — без записей (не ошибка)
- `anchor_date` — опорная дата (по умолчанию сегодня); границы периода в UTC

### Итоги тренировок (отчёты)

| Кто       | Данные                       | Период                |
| --------- | ---------------------------- | --------------------- |
| Тренер    | отчёты связанного спортсмена | последние 7 / 30 дней |
| Спортсмен | свои отчёты                  | последние 7 / 30 дней |

- В ответе: `difficulty_rating`, `mood_rating`, дата, ссылка на запланированную тренировку, опциональный комментарий
- Сортировка по дате (новые сверху)

### Доступ и безопасность

- Тренер — только связанные спортсмены (`VIEW_LINKED_ATHLETE_DATA` + guard)
- Спортсмен — только свои данные
- Чужой доступ — 403/404 без утечки

### Не входит

- Таблица результатов по упражнениям / дистанциям
- Графики динамики
- ИИ-ассистент

## Acceptance Criteria

- [x] Тренер видит план спортсмена на неделю и на месяц
- [x] Спортсмен видит свой план на неделю и на месяц
- [x] Тренер видит итоги тренировок спортсмена за неделю и за месяц
- [x] Спортсмен видит свои итоги за неделю и за месяц
- [x] Попытка доступа к чужим данным блокируется
- [x] API-ответы соответствуют контракту (MVP2-001)
- [x] Unit-тесты на правила доступа
- [x] Integration-тесты на типовые запросы по периодам
