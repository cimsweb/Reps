# MVP0-005 — UI

## Goal

Собрать пользовательский интерфейс MVP 0: регистрация, вход и личные кабинеты для администратора, тренера и спортсмена.

## Requirements

### Публичные страницы

- Форма регистрации: email, пароль, выбор роли (`coach` / `athlete`)
- Форма входа: email, пароль
- Редирект на личный кабинет после успешного входа
- Редирект на вход при отсутствии сессии на защищённых страницах

### Личные кабинеты

- **Admin:** список всех пользователей (email, роль, дата регистрации)
- **Coach:** приветствие и базовая информация аккаунта
- **Athlete:** приветствие и базовая информация аккаунта

### Общее

- Недоступные разделы скрыты по роли
- Ошибки валидации отображаются рядом с полями
- Состояния загрузки и пустые списки обработаны
- `npm run lint` и `npm run format:check` проходят

## Acceptance Criteria

- [x] Пользователь регистрируется и входит через UI
- [x] После входа открывается кабинет, соответствующий роли
- [x] Администратор видит список пользователей в UI
- [x] Тренер и спортсмен не видят admin-раздел
- [x] Неавторизованный пользователь не попадает в защищённые страницы
- [x] Ошибки API отображаются понятно
- [x] Основные сценарии покрыты UI/integration-тестами (если инфраструктура готова)

## Dependencies

- Backend API MVP 0 полностью работает
