# MVP0-004 — Search

## Goal

Дать ролям возможность получать нужные данные: администратор — список всех пользователей; тренер и спортсмен — данные своего личного кабинета.

> В контексте Reps «поиск» — запросы и фильтрация пользовательских данных, а не полнотекстовый поиск.

## Requirements

### Для администратора

- Список всех пользователей: email, роль, дата регистрации
- Пагинация при количестве записей > 20
- Сортировка по дате регистрации (новые первые)

### Для тренера и спортсмена

- Эндпоинт «текущий пользователь»: email, роль, id
- Данные личного кабинета доступны только владельцу сессии

### Безопасность

- Только `admin` получает список всех пользователей
- `coach` и `athlete` не имеют доступа к admin-эндпоинтам
- Ответы API соответствуют контракту (MVP0-001)

## Acceptance Criteria

- [x] Администратор видит список всех пользователей
- [x] Тренер и спортсмен видят только данные своего аккаунта
- [x] Попытка `coach` / `athlete` получить список пользователей возвращает 403
- [x] Пагинация работает корректно
- [x] Unit-тесты на правила доступа к запросам
- [x] Integration-тесты на типовые сценарии просмотра

## Dependencies

- [MVP0-003-indexing.md](MVP0-003-indexing.md) закрыт
- Роли и guards работают
