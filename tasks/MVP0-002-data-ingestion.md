# MVP0-002 — Data Ingestion

## Goal

Реализовать загрузку и сохранение учётных данных: регистрация нового пользователя и аутентификация при входе.

## Requirements

### Регистрация

- Принимать: email, пароль, роль (`coach` или `athlete`; `admin` — только через seed/миграцию)
- Валидация email (формат, уникальность)
- Валидация пароля (минимальная длина, сложность — зафиксировать в домене)
- Хеширование пароля перед сохранением; пароль в открытом виде не хранится

### Вход

- Принимать: email, пароль
- Проверять учётные данные; при успехе — создавать сессию / токен
- При неудаче — единообразная ошибка без утечки «email не найден» vs «неверный пароль»

### Общее

- Все операции через use cases в `application/`
- Ошибки валидации — явные доменные исключения
- Каждая операция записи логируется (см. MVP0-001)

## Acceptance Criteria

- [x] Пользователь регистрируется с валидным email и паролем
- [x] Дубликат email отклоняется
- [x] Слабый пароль отклоняется с понятной ошибкой
- [x] Пользователь входит с корректными учётными данными
- [x] Неверные учётные данные отклоняются без утечки информации
- [x] Пароль хранится только в виде хеша
- [x] Unit-тесты покрывают валидацию и доменные правила
- [x] Integration-тесты покрывают сохранение и вход через БД
