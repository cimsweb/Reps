# MVP 2.2 — ИИ-ассистент планов и отчётов

**Этап:** ИИ-усиление  
**Предшественник:** [mvp-2.1.md](mvp-2.1.md)  
**Следующий:** [mvp-3.md](mvp-3.md)

## Цель

ИИ ведёт себя как **хороший агент**: помогает тренеру развернуть короткий замысел в полноценную неделю, а спортсмену — оформить краткий ответ в содержательный отчёт. Человек всегда подтверждает результат перед сохранением.

## Проблема после MVP 2.1

Rule-based парсер понимает уже написанный текст, но не **создаёт** план из намерения. Спортсмену всё ещё нужно самому додумывать детали отчёта, если он написал только «нагрузка 5».

## Scope

### Для тренера

- [ ] Режим **«Создать с ИИ»**: краткий бриф → черновик недели
    - Пример брифа: «бег, зал, бег, отдых, бег длинный, вело, бег лёгкий» + дата начала
    - ИИ задаёт **уточняющие вопросы** (объём, зоны, акцент недели) при нехватке контекста
- [ ] Шаблоны недели: **бег / зал / вело / отдых** — из короткого списка типов
- [ ] Черновик в том же формате, что MVP 2.1 (текст + структура); тренер правит и сохраняет
- [ ] История диалога с агентом в рамках сессии создания плана

### Для спортсмена

- [ ] **ИИ-ассистент отчёта**: короткий ввод («нагрузка 5», «тяжёлые интервалы, настроение норм»)
- [ ] Агент задаёт **встречные вопросы**:
    - что было труднее всего?
    - как самочувствие после?
    - есть ли ссылка на Garmin?
- [ ] Итог: структурированный отчёт (`difficulty_rating`, `mood_rating`, `comment`, `garmin_url`) — спортсмен подтверждает отправку
- [ ] Не подменяет честность: финальные оценки выставляет спортсмен

### Общее

- [ ] `AIService` — изолированный адаптер в `infrastructure/`; domain не зависит от провайдера
- [ ] Лимиты: таймаут, макс. число сообщений в сессии, логирование без PII в проде
- [ ] Явная обработка сбоев ИИ («попробуйте позже» / ручной ввод из MVP 2.1)

## Вне scope

- Автономное сохранение без подтверждения пользователя
- ИИ анализирует нагрузку за месяц / алерты перетренированности
- Голосовой ввод
- Garmin API (MVP 3)

## Зависимости

- MVP 2.1 закрыт: текстовый ввод, парсер, расширенный отчёт, mobile UI
- Согласован провайдер LLM и переменные окружения (`AI_*` в `.env`)

## Критерии готовности

- Тренер из брифа «бег зал бег …» получает черновик недели, похожий на реальный пример тренера
- Агент задаёт минимум один уточняющий вопрос при неполном брифе
- Спортсмен вводит «нагрузка 5» — агент уточняет детали и предлагает заполненный отчёт
- Ни один план и ни один отчёт не сохраняются без явного действия пользователя
- При недоступности ИИ доступен fallback на MVP 2.1 (ручной / текстовый ввод)
- Диалоги и промпты покрыты unit-тестами (мок AIService); интеграционные — с записанными ответами
