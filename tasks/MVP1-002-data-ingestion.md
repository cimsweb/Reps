# MVP1-002 — Data Ingestion

## Goal

Реализовать загрузку и сохранение пользовательских данных MVP 1: приглашения, профиль спортсмена, личные рекорды, обратная связь по тренировке и ссылка на Garmin Connect.

## Requirements

### Приглашения

- Тренер отправляет приглашение спортсмену по email
- Спортсмен принимает или отклоняет приглашение
- Повторное приглашение уже связанному спортсмену запрещено

### Профиль спортсмена

- Поля: рост, вес, возраст, пол
- Только спортсмен редактирует свой профиль
- Обязательная валидация типов и диапазонов значений

### Личные рекорды

- Рекорд привязан к дистанции или упражнению
- Поля: тип, значение, единица измерения, дата
- Спортсмен создаёт, редактирует и удаляет свои рекорды

### Обратная связь по тренировке

- Спортсмен оставляет текстовую обратную связь после тренировки
- Можно прикрепить ссылку на отчёт в Garmin Connect
- Валидация URL ссылки на Garmin Connect

### Общее

- Все операции записи проходят через use cases в `application/`
- Ошибки валидации возвращаются явно, без generic-сообщений
- Каждая операция записи логируется (см. MVP1-001)

## Acceptance Criteria

- [x] Тренер может отправить приглашение по email; статус приглашения отслеживается
- [x] Спортсмен может принять приглашение; создаётся связь `CoachAthleteLink`
- [x] Спортсмен может заполнить и обновить профиль; невалидные данные отклоняются
- [x] Спортсмен может добавить, изменить и удалить личный рекорд
- [x] Спортсмен может оставить обратную связь с опциональной ссылкой на Garmin
- [x] Невалидная ссылка Garmin отклоняется с понятной ошибкой
- [x] Unit-тесты покрывают валидацию и доменные правила
- [x] Integration-тесты покрывают сохранение в БД
