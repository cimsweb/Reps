# MVP0-001 — Project Setup

## Goal

Подготовить технический фундамент MVP 0: структура проекта, домен аутентификации, контракты API, БД и логирование. После задачи можно приступать к регистрации, входу и ролям.

## Requirements

- Создать целевую структуру каталогов (`src/domain`, `application`, `infrastructure`, `interfaces`, `tests`)
- Описать доменные сущности:
    - `User` — пользователь системы
    - `Role` — роль: `admin`, `coach`, `athlete`
    - `Session` / токен аутентификации (концептуально)
- Зафиксировать value objects: `Email`, `PasswordHash`, `UserId`
- Определить интерфейсы репозиториев: `UserRepository`, `SessionRepository`
- Спроектировать схему БД Postgresql и миграции для пользователей и ролей
- Описать REST API: регистрация, вход, выход, текущий пользователь
- Настроить структурированное логирование:
    - регистрация и вход (успех / неудача)
    - проверка роли на защищённых маршрутах
    - ошибки валидации и авторизации
- Подготовить заготовки use cases без бизнес-логики
- Не реализовывать UI и формы — только каркас

## Acceptance Criteria

- [x] Структура каталогов соответствует DDD-правилам проекта
- [x] Доменные сущности и value objects описаны в `src/domain/`
- [x] Интерфейсы репозиториев определены; реализации — в `infrastructure/`
- [x] Миграции БД созданы и применяются на чистой базе
- [x] API-контракт аутентификации задокументирован
- [x] Логгер настроен; формат логов единообразный
- [x] Заготовки use cases существуют; smoke-тесты импорта проходят
- [x] `ruff`, `mypy`, `pytest` проходят
