# MVP3-001 — Project Setup

## Goal

Подготовить технический фундамент MVP 3: интеграция с Garmin API, домен синхронизации активностей, контракты и логирование.

## Requirements

- Описать доменные сущности:
    - `GarminConnection` — OAuth-токены спортсмена
    - `ImportedActivity` — активность, импортированная с Garmin
    - `SyncJob` — задача синхронизации (статус, время, ошибка)
- Определить интерфейс `GarminApiClient` в `domain/`
- Зафиксировать интерфейсы репозиториев для подключений и активностей
- Спроектировать схему БД и миграции
- Описать REST API: OAuth flow, запуск синхронизации, статус, список активностей
- Настроить логирование:
    - OAuth: успех / отказ / истечение токена
    - синхронизация: старт, завершение, ошибка, количество импортированных записей
    - токены и секреты не попадают в логи
- Подготовить заготовки use cases и фоновой задачи синхронизации
- Не реализовывать реальные вызовы Garmin — только каркас

## Acceptance Criteria

- [x] Доменные сущности описаны в `src/domain/`
- [x] `GarminApiClient` — интерфейс в domain; реализация — в `infrastructure/`
- [x] Миграции БД созданы и применяются
- [x] API-контракт OAuth и синхронизации задокументирован
- [x] Логгер настроен; секреты не логируются
- [x] Заготовки use cases и smoke-тесты проходят
- [x] `ruff`, `mypy`, `pytest` проходят

- Аналитика, таблицы и графики динамики работают
