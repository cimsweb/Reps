# MVP 2.1 — Удобный ввод планов и отчётов (без ИИ)

**Этап:** Улучшение UX  
**Предшественник:** [mvp-2.md](mvp-2.md)  
**Следующий:** [mvp-2.2.md](mvp-2.2.md)

## Цель

Сделать работу с планами и отчётами такой же естественной, как в таблице или мессенджере: тренер вставляет текст недели целиком, спортсмен — ссылку Garmin или короткую фразу. Интерфейс дружелюбен на **desktop и mobile**.

## Проблема MVP 2

Текущий структурный редактор (циклы → упражнения) не отражает реальный рабочий процесс:

- Тренер пишет план **свободным текстом** по дням (разминка, зоны, раунды, отдых, растяжка).
- Спортсмен отвечает **вставкой из Garmin** или короткой оценкой («нагрузка 6–7»), а не только слайдерами 0–10.

Пример недели тренера (8–14 июня): суставная разминка → интервалы в зонах → велостанок → силовой круг → длительный бег — всё в одном текстовом блоке на день.

Пример отчёта спортсмена: готовое сообщение Garmin Connect с URL, иногда только «нагрузка 6–7» или дополнение «50 минут».

## Scope

### Для тренера

- [ ] Три режима создания плана: **структурный** (как сейчас), **один день (текст)**, **неделя (текст)**
- [ ] Свободный ввод в `textarea`; **парсинг на backend** (rule-based, без LLM) в `PlannedWorkout` + циклы + упражнения
- [ ] **Превью** распознанной структуры перед сохранением; ручная правка после парсинга
- [ ] Сохранение **исходного текста** плана для повторного редактирования
- [ ] Просмотр плана в **читаемом виде** (как у тренера в таблице) + структурированный вид

### Для спортсмена

- [ ] Отчёт: поле **«вставьте как есть»** (Garmin, короткая оценка, смешанный текст)
- [ ] Автоизвлечение `garmin_url` и подсказка типа активности из текста Garmin
- [ ] Распознавание фраз вида «нагрузка 6–7» → предзаполнение `difficulty_rating`
- [ ] Слайдеры трудности/эмоций остаются; можно уточнить после вставки

### Общее

- [ ] **Mobile-first** вёрстка разделов тренировок: недельная сетка на desktop, карточки по дням на mobile
- [ ] Единые состояния: загрузка, пустой период, ошибка парсинга с понятным сообщением

## Вне scope

- LLM и генерация планов (MVP 2.2)
- Автоматический импорт Garmin API (MVP 3)
- Таблицы результатов, графики динамики
- Нативные мобильные приложения

## Зависимости

- MVP 2 полностью закрыт (backend + UI)
- MVP 1: связка coach ↔ athlete, профили

## Критерии готовности

- Тренер вставляет текст недели из таблицы/Excel и получает структурированный план после превью
- Тренер может создать один день тем же способом
- Спортсмен вставляет сообщение Garmin — URL и оценка подставляются автоматически
- Спортсмен может отправить только «нагрузка 6–7» — система предлагает рейтинг, отчёт сохраняется
- UI удобен на ширине 375px и на desktop
- Парсер покрыт unit-тестами на реальных примерах (неделя 8–14 июня, отчёты Garmin)

## Референсные примеры (для тестов парсера)

Сохранить в `tests/fixtures/training_text/` как эталонные входы:

- `week_2024-06-08.txt` — неделя 8–14 июня (7 колонок / 7 блоков по датам)
- `report_garmin_run.txt` — стандартное сообщение Garmin Connect с URL
- `report_load_short.txt` — «нагрузка 6-7»
