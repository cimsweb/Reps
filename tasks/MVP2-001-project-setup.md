# MVP2-001 — Project Setup

## Goal

Подготовить технический фундамент MVP 2: домен планов тренировок и отчётов о выполнении, контракты API, схема БД и логирование. Без UI и без бизнес-логики записи.

## Requirements

### Доменные сущности

- `TrainingPlan` — план тренировок тренера для спортсмена (период: день или неделя)
- `PlannedWorkout` — одна запланированная тренировка на дату
- `WorkoutType` — тип: `run` (бег), `bike` (велосипед), `gym` (зал)
- `WorkoutCycle` — цикл внутри тренировки (например, разминка, основная часть, заминка)
- `WorkoutExercise` — упражнение или интервал в цикле (название, параметры, порядок в списке)
- `WorkoutCompletionReport` — отчёт спортсмена по выполненной тренировке

### Value objects

- `DifficultyRating` — целое 0–10 (10 — максимальная трудность)
- `MoodRating` — целое 0–10 (10 — максимально позитивное состояние)
- Даты плана: `planned_date`, границы периода плана

### Инфраструктура

- Интерфейсы репозиториев в `domain/` (план, тренировки, циклы, упражнения, отчёты)
- Миграции БД для сущностей MVP 2
- REST API описан в документации (создание плана, отчёт, чтение по периодам — контракты без реализации)
- Структурированное логирование:
    - создание / изменение плана
    - отправка отчёта спортсменом
    - ошибки валидации и авторизации
- Заготовки use cases в `application/` (`NotImplementedError` или аналог MVP 1)

### Не входит в задачу

- ИИ и `AIService`
- Реализация форм и UI
- Агрегация за неделю / месяц (MVP2-003, MVP2-004)
