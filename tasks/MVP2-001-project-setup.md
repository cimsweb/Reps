# MVP2-001 — Project Setup

## Goal

Подготовить технический фундамент MVP 2: домен ИИ-ассистента и аналитики, контракты API, инфраструктура хранения программ тренировок и логирования.

## Requirements

- Описать доменные сущности:
    - `TrainingProgram` — программа тренировок (черновик / подтверждённая)
    - `WorkoutReport` — развёрнутый отчёт о тренировке
    - `PerformanceMetric` — показатель для таблицы и графика
    - `MetricSnapshot` — значение показателя на дату
- Определить интерфейс `AIService` в `domain/` (без привязки к провайдеру)
- Зафиксировать интерфейсы репозиториев для программ, отчётов и метрик
- Спроектировать схему БД и миграции
- Описать REST API: генерация программы, генерация отчёта, метрики, график
- Настроить логирование:
    - запросы к ИИ (без персональных данных в логах)
    - подтверждение / отклонение черновика пользователем
    - ошибки ИИ-провайдера
- Подготовить заготовки use cases без бизнес-логики
- Не реализовывать UI и вызовы ИИ — только каркас

## Acceptance Criteria

- [ ] Доменные сущности описаны в `src/domain/`
- [ ] `AIService` — интерфейс в domain; реализация — в `infrastructure/`
- [ ] Миграции БД созданы и применяются
- [ ] API-контракт задокументирован
- [ ] Логгер настроен; чувствительные данные не попадают в логи
- [ ] Заготовки use cases и smoke-тесты проходят
- [ ] `ruff`, `mypy`, `pytest` проходят

## Dependencies

- [mvp-1.md](mvp-1.md) полностью закрыт
- Связка тренер ↔ спортсмен, профили, рекорды, feedback работают
