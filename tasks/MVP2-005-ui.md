# MVP2-005 — UI

## Goal

Собрать интерфейс MVP 2: тренер создаёт и просматривает планы, спортсмен сдаёт отчёты и смотрит план и итоги. **Без ИИ** — только формы и календарные представления.

## Requirements

### Кабинет тренера

**Создание плана**

- Выбор спортсмена из списка связанных
- Режим: план на **день** или на **неделю**
- Для каждой тренировки: тип (бег / велосипед / зал)
- Редактор циклов и списков упражнений (добавить цикл, добавить упражнение в список, порядок)
- Сохранение с обработкой ошибок API

**Просмотр**

- Календарь / список: план на **неделю** и на **месяц** вперёд (переключение `anchor_date`)
- Итоги тренировок выбранного спортсмена за **неделю** и за **месяц**
- Отображение оценок трудности и эмоций в итогах

### Кабинет спортсмена

**Отчёт**

- Список предстоящих / выполненных тренировок из плана
- Форма отчёта: шкала или input 0–10 для трудности и эмоций, опциональный комментарий
- Одна отправка на тренировку; уже сданная — без повторной формы

**Просмотр**

- План на **неделю** и на **месяц**
- Свои итоги за **неделю** и за **месяц**

### Общее

- Разделы MVP 2 доступны только соответствующей роли
- Состояния загрузки, пустые периоды, ошибки API — как в MVP 1
- Адаптивная вёрстка (desktop + mobile)
- `npm run lint` и `npm run format:check` проходят

### Не входит

- Кнопки «Сгенерировать с ИИ»
- Таблица результатов и графики динамики

## Acceptance Criteria

- [x] Тренер создаёт план на день и на неделю через UI
- [x] Тренер видит план спортсмена на неделю и месяц
- [x] Тренер видит итоги спортсмена за неделю и месяц
- [x] Спортсмен отправляет отчёт с оценками 0–10 через UI
- [x] Спортсмен видит план и свои итоги за неделю и месяц
- [x] Ошибки API отображаются понятно
- [x] `npm run lint` и `npm run format:check` проходят
- [x] Основные сценарии проверены вручную (e2e — при появлении инфраструктуры)

- Backend API MVP 2 полностью работает
- Кабинеты coach / athlete из MVP 1 существуют
