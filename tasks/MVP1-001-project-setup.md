# MVP1-001 — Project Setup

## Goal

Подготовить технический фундамент для MVP 1: доменные сущности, контракты API, инфраструктура хранения и логирования. После этой задачи команда может параллельно работать над загрузкой данных и интерфейсом.

## Requirements

- Описать доменные сущности MVP 1:
    - `Invitation` — приглашение спортсмена по email
    - `CoachAthleteLink` — связь тренер ↔ спортсмен
    - `AthleteProfile` — рост, вес, возраст, пол
    - `PersonalRecord` — рекорд на дистанции или в упражнении
    - `WorkoutFeedback` — обратная связь по итогу тренировки
    - `GarminReportLink` — ссылка на отчёт в Garmin Connect
- Зафиксировать интерфейсы репозиториев в `domain/`, без привязки к фреймворку
- Спроектировать схему БД и миграции для сущностей MVP 1
- Описать REST API (эндпоинты, request/response) в документации
- Настроить структурированное логирование:
    - создание приглашения
    - принятие/отклонение приглашения
    - сохранение профиля, рекорда, обратной связи
    - ошибки валидации и авторизации
- Подготовить заготовки use cases в `application/` без бизнес-логики
- Не реализовывать пользовательские сценарии — только каркас

## Acceptance Criteria

- [ ] Доменные сущности и value objects описаны в `src/domain/`
- [ ] Интерфейсы репозиториев определены, реализации — в `infrastructure/`
- [ ] Миграции БД созданы и применяются на чистой базе
- [ ] API-контракт задокументирован (README или отдельный файл)
- [ ] Логгер настроен; формат логов единообразный (уровень, событие, user_id, timestamp)
- [ ] Заготовки use cases существуют и покрыты smoke-тестами импорта
- [ ] Все изменения проходят `ruff`, `mypy`, `pytest`

## Dependencies

- Работают роли `coach` и `athlete`
- Аутентификация и авторизация стабильны
