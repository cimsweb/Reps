# MVP3-003 — Indexing

## Goal

Организовать хранение и связь импортированных активностей со спортсменом для построения графика успехов и аналитики.

> В контексте Reps «индексация» — привязка активностей к атлету, дедупликация и подготовка данных к запросам.

## Requirements

### Связи

- `ImportedActivity` привязана к `athlete_id`
- Активность связана с `GarminConnection` источника
- Метрики активности доступны для графика успехов

### Агрегация

- Активности группируются по типу: бег, трейл, OCR-связанные
- Ключевые метрики: дистанция, время, темп, набор высоты (если есть)
- Данные упорядочены по дате для временных рядов

### Производительность

- Индексы БД на `athlete_id`, `garmin_activity_id`, `activity_date`
- Уникальный индекс на `garmin_activity_id`
- Запрос активностей за период без N+1

## Acceptance Criteria

- [x] Каждая активность однозначно принадлежит спортсмену
- [x] Дубликаты исключены на уровне БД
- [x] Метрики активности доступны для агрегации
- [x] Индексы БД созданы и задокументированы
- [x] Unit-тесты на принадлежность и дедупликацию
- [x] Integration-тесты на агрегацию за период
