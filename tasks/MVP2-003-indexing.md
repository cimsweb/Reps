# MVP2-003 — Indexing

## Goal

Организовать агрегацию показателей и подготовку временных рядов для таблицы результатов и графика динамики.

> В контексте Reps «индексация» — группировка метрик по атлету, типу и дате для быстрых запросов аналитики.

## Requirements

### Агрегация метрик

- Метрики группируются по спортсмену
- Внутри спортсмена — по типу: дистанция / упражнение
- `MetricSnapshot` упорядочены по дате для построения графика

### Связи

- Программа тренировок привязана к тренеру и целевому спортсмену
- Отчёт привязан к спортсмену и опционально к feedback из MVP 1
- Тренер видит метрики только своих спортсменов

### Производительность

- Индексы БД на `athlete_id`, `metric_type`, `recorded_at`
- Запрос временного ряда без N+1
- Кэширование — вне scope MVP 2

## Acceptance Criteria

- [ ] Метрики однозначно принадлежат спортсмену
- [ ] Временной ряд строится из упорядоченных `MetricSnapshot`
- [ ] Тренер получает метрики только своих спортсменов
- [ ] Индексы БД созданы и задокументированы
- [ ] Unit-тесты на агрегацию и принадлежность
- [ ] Integration-тесты на построение ряда данных
