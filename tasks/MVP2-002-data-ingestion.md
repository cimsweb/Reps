# MVP2-002 — Data Ingestion

## Goal

Реализовать загрузку данных для ИИ и аналитики: входные параметры программы тренировок, черновики отчётов и фиксация показателей для таблиц и графиков.

## Requirements

### ИИ — программа тренировок (тренер)

- Тренер задаёт параметры: цель, длительность цикла, уровень атлета, ограничения
- ИИ возвращает черновик `TrainingProgram`
- Тренер редактирует и подтверждает программу перед сохранением
- Неподтверждённый черновик не считается финальной программой

### ИИ — отчёт о тренировке (спортсмен)

- Спортсмен задаёт краткие заметки о тренировке
- ИИ расширяет в развёрнутый `WorkoutReport`
- Спортсмен редактирует и подтверждает отчёт перед сохранением

### Метрики

- Показатели берутся из личных рекордов и feedback MVP 1
- Ручное добавление метрики спортсменом — если нет в MVP 1 данных
- Валидация: тип метрики, значение, единица, дата

### Общее

- Все записи через use cases
- Таймаут и ошибка ИИ обрабатываются явно
- Операции логируются (см. MVP2-001)

## Acceptance Criteria

- [ ] Тренер получает черновик программы от ИИ и может его отредактировать
- [ ] Программа сохраняется только после подтверждения тренером
- [ ] Спортсмен получает черновик отчёта от ИИ и может его отредактировать
- [ ] Отчёт сохраняется только после подтверждения спортсменом
- [ ] Метрики валидируются и сохраняются
- [ ] Сбой ИИ не ломает приложение; пользователь видит понятную ошибку
- [ ] Unit-тесты на валидацию и правило «черновик → подтверждение»
- [ ] Integration-тесты с mock `AIService`
