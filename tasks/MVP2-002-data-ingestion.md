# MVP2-002 — Data Ingestion

## Goal

Реализовать запись данных MVP 2: тренер создаёт план тренировок вручную (день / неделя), спортсмен отправляет отчёт о выполнении с оценками трудности и эмоций.

> **Без ИИ** — только формы и валидация.

## Requirements

### План тренировок (тренер)

- Тренер создаёт план только для **связанного** спортсмена (`CoachAthleteLink` из MVP 1)
- Режимы:
    - **На один день** — одна или несколько `PlannedWorkout` на выбранную дату
    - **На неделю** — набор тренировок на 7 дней (с понедельника или с выбранной даты — зафиксировать в API)
- Для каждой тренировки:
    - тип: бег / велосипед / зал
    - один или несколько **циклов** (`WorkoutCycle`)
    - внутри цикла — **список упражнений** (`WorkoutExercise`): название, описание/параметры (дистанция, время, повторения — текстовые или структурированные поля по контракту MVP2-001)
- Тренер может **редактировать** и **удалять** ещё не прошедшие планы (правило зафиксировать в домене)
- Повторное создание на ту же дату — явная ошибка или замена (зафиксировать в API)

### Отчёт о тренировке (спортсмен)

- Спортсмен создаёт отчёт по **запланированной** тренировке (привязка к `PlannedWorkout`)
- Обязательные поля:
    - `difficulty_rating` — 0–10 (10 — очень тяжело)
    - `mood_rating` — 0–10 (10 — «кайфую»)
- Опционально: текстовый комментарий (краткий)
- Один отчёт на одну запланированную тренировку (без дублей)
- Только спортсмен, которому назначена тренировка

### Общее

- Все операции записи — через use cases в `application/`
- Валидация диапазонов 0–10, типов тренировок, обязательных полей
- Ошибки домена — явные, без generic-сообщений
- Операции логируются (см. MVP2-001)

## Acceptance Criteria

- [x] Тренер создаёт план на день для связанного спортсмена
- [x] Тренер создаёт план на неделю с несколькими типами тренировок
- [x] План содержит циклы и списки упражнений
- [x] Спортсмен отправляет отчёт с `difficulty_rating` и `mood_rating` (0–10)
- [x] Невалидные оценки и чужие тренировки отклоняются
- [x] Unit-тесты на валидацию и доменные правила
- [x] Integration-тесты на сохранение в БД
