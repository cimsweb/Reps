# MVP3-002 — Data Ingestion

## Goal

Реализовать загрузку активностей из Garmin API: OAuth-авторизация, получение данных, валидация и сохранение.

## Requirements

### OAuth

- Спортсмен инициирует подключение Garmin через OAuth 2.0
- Токены сохраняются в `GarminConnection`; обновление refresh token
- Отключение интеграции удаляет токены

### Импорт активностей

- Фоновая задача запрашивает новые активности с момента последней синхронизации
- Валидация: тип активности, дата, дистанция, длительность, пульс (если есть)
- Дубликаты не создаются (идемпотентность по `garmin_activity_id`)
- Ручные ссылки Garmin из MVP 1 сохраняются; автоматический импорт их дополняет

### Обработка ошибок

- Истёкший токен → статус «требуется переподключение»
- Ошибка API → `SyncJob` в статусе `failed` с описанием
- Частичный импорт логируется; успешные записи не откатываются

## Acceptance Criteria

- [x] Спортсмен подключает Garmin через OAuth
- [x] Активности импортируются автоматически
- [x] Дубликаты не создаются при повторной синхронизации
- [x] Невалидные записи отклоняются с логированием
- [x] Истёкший токен обрабатывается явно
- [x] Unit-тесты на валидацию и идемпотентность
- [x] Integration-тесты с mock `GarminApiClient`
- [x] `ruff`, `mypy`, `pytest` проходят
