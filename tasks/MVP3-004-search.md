# MVP3-004 — Search

## Goal

Дать тренеру и спортсмену доступ к импортированным активностям и данным для графика успехов на основе реальных показателей с устройства.

> В контексте Reps «поиск» — запросы активностей, фильтрация по периоду и типу, выборка для графика.

## Requirements

### Список активностей

- Спортсмен видит свои импортированные активности
- Тренер видит активности своих спортсменов
- Фильтры: период, тип активности
- Пагинация при > 20 записей

### График успехов

- Строится на данных `ImportedActivity`, не на ручном вводе
- Метрики: дистанция, темп, набор высоты — по доступности данных
- Период: 30 / 90 / 365 дней / всё время

### Статус синхронизации

- Последняя успешная синхронизация: дата и время
- Текущий статус: `idle` / `running` / `failed` / `reconnect_required`
- При `failed` — краткое описание ошибки для пользователя

### Безопасность

- Тренер видит активности только своих спортсменов
- Чужие данные — 403/404

## Acceptance Criteria

- [x] Спортсмен видит список своих активностей с фильтрами
- [x] Тренер видит активности своих спортсменов
- [x] График успехов строится на импортированных данных
- [x] Статус синхронизации отображается корректно
- [x] API-ответы соответствуют контракту (MVP3-001)
- [x] Unit-тесты на правила доступа
- [x] Integration-тесты на запросы активностей и графика
